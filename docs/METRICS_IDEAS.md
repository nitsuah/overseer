# Developer Health Metrics Ideas

1.  **Code Coverage**: Percentage of code covered by tests.
2.  **Testing Status**: Passing/Failing status of the latest CI run.
3.  **Linting Errors**: Number of linting errors/warnings.
4.  **Pre-commit Hooks**: Presence and configuration of pre-commit hooks (e.g., husky).
5.  **Dependency Freshness**: Number of outdated dependencies.
6.  **Security Vulnerabilities**: Number of known vulnerabilities in dependencies (npm audit).
7.  **Bus Factor**: Estimated number of developers who know the codebase well.
8.  **Documentation Coverage**: Percentage of public API documented.
9.  **PR Turnaround Time**: Average time to close a PR.
10. **Issue Resolution Time**: Average time to close an issue.
11. **Last Commit Date**: Freshness of the codebase.
12. **Busyness**: Commit frequency over the last 30 days.
13. **Tech Debt Ratio**: Estimated technical debt based on TODOs/FIXMEs.
14. **CI/CD Pipeline Duration**: Average build/test time.
15. **License Compliance**: Check for valid and compatible licenses.
16. **Code Complexity**: Cyclomatic complexity metrics.
17. **Bundle Size**: Size of the production bundle (for web apps).
18. **Accessibility Score**: Lighthouse accessibility score.
19. **Performance Score**: Lighthouse performance score.
20. **SEO Score**: Lighthouse SEO score.
